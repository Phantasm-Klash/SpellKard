- `client_ui_smoke_test.gd`: focused headless menu-experience check. It waits for Control layout to settle, then verifies the home four-button lobby, four primary status cards, compact overview cards, parent/Home-only quick navigation, no visible control overlaps, no player-visible debug labels, and the play/match/certification/settings/community/friends/social/promotion surfaces named in the client plan.
- `balance_simulation_check.gd`: headless deterministic balance gate that runs current pattern, active-deck, character, and bullet-visual rules across seeds, prints the aggregate summary, and fails on blocking readability/performance/hit-rate/card-usage regressions while preserving score-spread warnings for tuning review.
- `boss_pattern_catalog_check.gd`: headless catalog gate that verifies the Boss pattern taxonomy is covered by stage patterns, the executable Boss-type matrix covers aimed shots, n-way fans, rotating rings, spirals, curtains/walls, warning/fire lasers, split/branching carriers, delayed blossoms, homing, orbital/lattice releases, stream/weave lanes, seeded randomness, phase scripts, and spellcard timeout/enrage rules. Each catalog type must emit deterministic bullets through `BulletPatternLibrary.emit_pattern`, seeded patterns must change with seed, shaped lasers must expose line/polyline metadata plus continuous-graze cooldowns, delayed/carrier behaviors must spawn child bullets through `BulletEngine.step_bullets`, Boss spellbooks must emit valid timeout/enrage phase scripts, deterministic phase preview/export signatures and compact golden digests must reproduce for the same seed, golden fixtures must pin sample ticks, sample emit counts, max emit, cap, headroom, and within-budget status, Pattern Lab must expose catalog/phase coverage rows with canonical phase-script budget fields and preview sample-window metadata, replay-list metadata must preserve spellbook preview export/digest/sample-window/sample-emit-count/budget data while marking local previews as non-authoritative and rejecting over-budget, stale digest, stale sample-window, stale sample emit-count, bad sample-count, bad sample emit-count, and missing-sample preview rows, full-duration spellbook scans must respect each phase script's own bullet cap and reject tight-cap regressions, and open-source migration recipes/adapters must include license/provenance while remaining mechanism-only references rather than direct copied content.
- `latency_matrix_check.gd`: headless multiplayer latency matrix gate for 30/80/150/250ms profiles, packet loss, jitter, short reconnect, input-delay bounds, snapshot correction metrics, perceived hit mismatch, battle royale deadline stability, Boss transfer idempotency, and certification server-result display. It runs against isolated model instances so the check does not leave queue, network-match, or Boss-transfer state behind in the main scene.
- `asset_manifest_check.gd`: headless asset gate that validates `godot/assets/asset_manifest.json`, base theme manifest fields, allowed licenses, and manifest coverage for production files under `assets` and `themes`.
- `gensoulkyo_live_http_check.gd`: live integration check for the Godot `HTTPRequest` transport. It expects Gensoulkyo at `127.0.0.1:7350`, then exercises anonymous login, bootstrap server inventory/decks/chests, explicit inventory sync, server deck save/deck sync, server chest sync/open with audit/result projection, server card upgrade projection, heartbeat presence for queue/running/disconnected states, two-player same-stage queue formation, server-confirmed loadout propagation, network-screen room-code creation/cancellation/join, ticket polling, ready, input, authoritative server bullet delta intake, server active-card projection, cursor event polling, disconnect/reconnect full-snapshot restore, full snapshot sync, post-settlement rematch waiting/found plus new-match ready/start, server-validated world Boss card-transfer mode actions, settlement, server replay audit-record read, and activity claim.
